## 概述

MetadataBinding（记忆链接）指由第三方应用提供[App Linking链接](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/applinking-introduction)，系统将当前用户浏览的内容与App Linking链接进行关联并保存的功能。

详细的接口介绍请参考[MetadataBinding接口](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-awareness-metadatabinding)。

## 场景介绍

第三方应用可使用记忆链接功能，将App Linking链接映射到调用接口的系统应用或服务。例如，用户在【电商应用】中浏览某个商品时，截图保存了该商品的图片，系统将记录图片与【电商应用】提供的App Linking链接的映射关系。当用户再次浏览该图片时，系统会提醒用户是否需要返回【电商应用】查看商品详情。

## 接口说明

- 本模块首批接口从API version 18开始支持。后续版本的新增接口，采用上角标单独标记接口的起始版本。
- 本模块支持记忆链接的功能。

  展开

| 接口名 | 描述 |
| --- | --- |
| submitMetadata (metadata: string): void; | 第三方应用将待编码的App Linking链接传递给多模态融合感知服务，该服务决定适当时机将内容传递给调用编码接口的系统应用。 |
| on (type: 'operationSubmitMetadata', bundleName: string, callback: Callback<number>): void; | 订阅系统事件以获取编码内容，应用注册回调，事件发生时回传编码内容。 |
| off (type: 'operationSubmitMetadata', bundleName: string, callback?: Callback<number>): void; | 取消订阅系统获取编码内容的事件。取消注册回调接口。 |

## 约束与限制

- App Linking链接的最大字节数为99。

## 开发步骤

1. 导入模块。

 收起自动换行深色代码主题复制

```
import { metadataBinding } from '@kit.MultimodalAwarenessKit' ; import { BusinessError } from '@kit.BasicServicesKit' ; import { Callback } from '@kit.BasicServicesKit' ;
```
2. 定义记忆服务回调及包名, 函数接收回传编码的内容。

 收起自动换行深色代码主题复制

```
let callback : Callback < number > = ( event : number ) => {}; let bundleName : string = '' ;
```
3. 订阅记忆服务。

 收起自动换行深色代码主题复制

```
try { metadataBinding. on ( 'operationSubmitMetadata' , bundleName, callback); console . info ( "on succeeded" ); } catch (err) { let error = err as BusinessError ; console . error ( "Register event error and err code is " + error. code ); }
```
4. 提供App Linking链接。

 收起自动换行深色代码主题复制

```
let metadata : string = "" ; try { metadataBinding. submitMetadata (metadata); } catch (err) { let error = err as BusinessError ; console . error ( "Submit metadata error and err code is " + error. code ); }
```
5. 取消订阅记忆服务。

 收起自动换行深色代码主题复制

```
try { metadataBinding. off ( 'operationSubmitMetadata' , bundleName, callback); console . info ( "off succeeded" ); } catch (err) { let error = err as BusinessError ; console . error ( "Unregister event error and err code is " + error. code ); }
```
